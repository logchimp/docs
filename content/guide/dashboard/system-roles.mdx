These roles are automatically created and managed by LogChimp. They cannot be edited or deleted.

Next to each system role, a "Computer" icon is displayed.

## List of System Roles

These are the following roles generated by the LogChimp.

| Role Name | Description                                                 |
|-----------|-------------------------------------------------------------|
| @everyone | This role is automatically assigned to newly created users. |

More roles will be added in the future.

## FAQ

<Accordions type="multiple">
	<Accordion title="How do I distinguish between system roles and custom roles?">
		The best way to find `@everyone` 🖥️ named system role is to see if a "Computer" icon is displayed next to it.
	</Accordion>

	<Accordion title="Why are there two roles named @everyone?">
		TLDR; This was just a pure coincidence.

		The `@everyone` role was created in the early days of the LogChimp with an idea to build a scalable role-permission system in LogChimp.

		We came a long way, and now based upon our learnings, we implemented a system called "System Roles".

		It will be superseded by a new System role named `@everyone` 🖥️ which will be used to assign to newly created users.
	</Accordion>

	<Accordion title="What will happen to existing @everyone role?">
		The `@everyone` role WILL NOT be deleted for backward compatibility.
	</Accordion>

	<Accordion title="What will happen to existings users with @everyone role?">
		Your existing users with `@everyone` role assigned are not affected.

		They will continue to have the same permissions as before.
	</Accordion>
</Accordions>
